## Hacker News Post

### Title
Launch HN: BuzzInvitly – Modern event invitations with RSVP tracking

### Post

Hi HN! I'm excited to share BuzzInvitly (https://buzzinvitly.com), a platform I built for creating event invitations and managing RSVPs.

**The problem:**

I've organized many events (personal and professional), and managing invitations was always frustrating. Using a mix of Canva, Google Sheets, email, and texts felt archaic. I wanted something modern and integrated.

**The solution:**

BuzzInvitly combines:
• Template-based invitation design (with full customization)
• Guest list management (manual + CSV import)
• Multi-channel delivery (email, SMS, shareable links)
• Real-time RSVP tracking
• Analytics with a "Buzz Score" metric

**Tech stack:**

• Frontend: Next.js 16 (App Router), React 19, TypeScript, Tailwind CSS
• Backend: Node.js, Prisma ORM, PostgreSQL
• Canvas: Fabric.js for the design editor
• Charts: Recharts for analytics
• Infrastructure: Vercel, Supabase
• Testing: Vitest, Playwright (80%+ coverage)

Performance was a priority—we achieved:
• 90+ Lighthouse score
• <2s page loads
• 70% faster DB queries with 25+ indexes
• In-memory caching layer

**Interesting technical challenges:**

1. **Fabric.js integration**: Building a user-friendly canvas editor was tricky. Balancing power with simplicity required careful UX decisions.

2. **Email deliverability**: Ensuring invitations don't land in spam involved proper SPF/DKIM setup and careful template design.

3. **RSVP tracking**: Real-time updates without websockets—we use optimistic UI updates + periodic polling.

4. **Buzz Score algorithm**: Weighs response rate (30%), attendance (30%), response speed (20%), engagement (10%), and social shares (10%) to create a 0-100 excitement score.

**Business model:**

Freemium with coins:
• FREE: 5 events/month, 50 guests/event
• PRO: $9.99/month - Unlimited events
• Pay-per-use: Coins for email (1 coin = $0.10)

**What's next:**

• WhatsApp integration
• Calendar sync (Google, Outlook)
• Video invitations
• Advanced analytics

**Questions for HN:**

1. What features would make this a must-have for you?
2. How do you currently handle event invitations?
3. Any concerns about privacy/data handling?

I'd love your feedback, technical questions, or feature suggestions!
